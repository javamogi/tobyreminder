# Apple Reminders Web - Spec

## 1. 프로젝트 개요

Apple Reminders 앱의 핵심 기능을 웹 브라우저에서 사용할 수 있도록 구현하는 프로젝트.
macOS/iOS의 Reminders 앱과 유사한 UI/UX를 제공하며, 리마인더 생성·관리·검색 등 핵심 기능에 집중한다.

### 목표
- Apple Reminders의 핵심 워크플로우를 웹에서 재현
- 직관적인 3-column 레이아웃으로 빠른 탐색과 관리
- 심플하고 깔끔한 Apple 디자인 스타일 유지

## 3. 핵심 기능

### 3.1 리스트 관리
- 리마인더 리스트 생성 / 수정 / 삭제
- 리스트별 색상 및 아이콘 지정
- 리스트 순서 변경 (드래그 앤 드롭)
- 리스트 내 리마인더 개수 표시

### 3.2 리마인더 CRUD
- 리마인더 생성 (제목 필수, 메모 선택)
- 리마인더 수정 / 삭제
- 완료 체크 토글
- 마감일(due date) 설정
- 마감 시간 설정 (선택)
- 반복 설정 (매일/매주/매월/매년/사용자 정의)
- 하위 리마인더 (subtask) 지원

### 3.3 우선순위 & 플래그
- 우선순위 4단계: 없음 / 낮음 / 보통 / 높음
- 플래그(Flag) 토글 — 중요 리마인더 마킹

### 3.4 스마트 리스트
자동 필터링되는 시스템 리스트:

| 스마트 리스트 | 필터 조건 |
|--------------|----------|
| **오늘 (Today)** | 마감일이 오늘인 미완료 리마인더 |
| **예정 (Scheduled)** | 마감일이 설정된 미완료 리마인더 (날짜별 그룹) |
| **전체 (All)** | 모든 미완료 리마인더 |
| **플래그 지정됨 (Flagged)** | 플래그가 설정된 미완료 리마인더 |
| **완료됨 (Completed)** | 완료된 리마인더 |

### 3.5 검색
- 리마인더 제목 및 메모 내용 키워드 검색
- 검색 결과 실시간 필터링

### 3.6 정렬
- 리스트 내 리마인더 수동 정렬 (드래그 앤 드롭)
- 마감일 / 생성일 / 우선순위 / 제목 기준 자동 정렬

## 4. 데이터 모델

### 4.1 ERD

```
┌──────────────────────┐       ┌──────────────────────────────┐
│    reminder_list      │       │         reminder              │
├──────────────────────┤       ├──────────────────────────────┤
│ id (PK)              │       │ id (PK)                      │
│ name                 │       │ list_id (FK)                 │
│ color                │       │ parent_id (FK, self-ref)     │
│ icon                 │       │ title                        │
│ display_order        │       │ notes                        │
│ created_at           │       │ is_completed                 │
│ updated_at           │       │ completed_at                 │
└──────────┬───────────┘       │ due_date                     │
           │                   │ due_time                     │
           │  1 : N            │ priority (NONE/LOW/MED/HIGH) │
           └───────────────────│ is_flagged                   │
                               │ recurrence_rule              │
                               │ display_order                │
                               │ created_at                   │
                               │ updated_at                   │
                               └──────────────────────────────┘
```

### 4.2 엔티티 상세

**ReminderList**
| 필드 | 타입 | 설명 |
|------|------|------|
| id | Long (PK) | 자동 생성 |
| name | String | 리스트 이름 (필수) |
| color | String | HEX 색상 코드 (기본: #007AFF) |
| icon | String | 아이콘 식별자 (기본: list.bullet) |
| displayOrder | Integer | 표시 순서 |
| createdAt | LocalDateTime | 생성 시각 |
| updatedAt | LocalDateTime | 수정 시각 |

**Reminder**
| 필드 | 타입 | 설명 |
|------|------|------|
| id | Long (PK) | 자동 생성 |
| list | ReminderList (FK) | 소속 리스트 |
| parent | Reminder (FK, nullable) | 상위 리마인더 (subtask용) |
| title | String | 제목 (필수) |
| notes | String (nullable) | 메모 |
| isCompleted | Boolean | 완료 여부 (기본: false) |
| completedAt | LocalDateTime (nullable) | 완료 시각 |
| dueDate | LocalDate (nullable) | 마감일 |
| dueTime | LocalTime (nullable) | 마감 시간 |
| priority | Priority (Enum) | NONE, LOW, MEDIUM, HIGH |
| isFlagged | Boolean | 플래그 여부 (기본: false) |
| recurrenceRule | String (nullable) | 반복 규칙 (DAILY/WEEKLY/MONTHLY/YEARLY/커스텀) |
| displayOrder | Integer | 표시 순서 |
| createdAt | LocalDateTime | 생성 시각 |
| updatedAt | LocalDateTime | 수정 시각 |

## 6. UI/UX 요구사항

> Apple Reminders (macOS Sequoia / iOS 18) 앱의 외관과 동작을 최대한 충실히 재현한다.

### 6.1 전체 레이아웃

macOS Reminders 앱과 동일한 **사이드바 + 콘텐츠** 구조:

```
┌─────────────────────┬──────────────────────────────────────────┐
│      Sidebar         │            Content Area                  │
│  (280px, 리사이즈)    │                                          │
│                     │  ┌─ 툴바 ──────────────────────────────┐  │
│  🔍 검색 필드        │  │  리스트 이름          정렬 ▾  ···   │  │
│                     │  └────────────────────────────────────┘  │
│  ┌ 스마트 리스트 ──┐ │                                          │
│  │ 📅 오늘    3    │ │  ○ 장보기                                │
│  │ 📋 예정    12   │ │    우유, 빵, 계란                        │
│  │ 📁 전체    25   │ │    📅 오늘  ⚑                           │
│  │ 🚩 플래그   2   │ │  ─────────────────────────────────────  │
│  │ ✅ 완료됨  48   │ │  ○ 회의 준비                             │
│  └────────────────┘ │    발표 자료 검토                         │
│                     │    📅 내일  !! 높음                       │
│  내 리스트           │  ─────────────────────────────────────  │
│  ● 개인        8    │  ○ 운동하기                               │
│  ● 업무       14    │                                          │
│  ● 쇼핑        3    │                                          │
│                     │                                          │
│                     │  + 리마인더 추가                           │
│  ─────────────────  │                                          │
│  + 리스트 추가       │                                          │
└─────────────────────┴──────────────────────────────────────────┘
```

### 6.2 사이드바 상세

**검색 필드**
- 사이드바 최상단에 위치, Apple 스타일 둥근 검색바 (배경: `rgba(0,0,0,0.06)`, 라운드: 10px)
- 돋보기 아이콘 + placeholder "검색"
- 포커스 시 배경 밝아지며 테두리 강조
- 입력 시 콘텐츠 영역이 검색 결과 리스트로 전환

**스마트 리스트 그리드**
- 2×2 또는 2×3 그리드 형태의 카드 (실제 Apple Reminders와 동일)
- 각 카드: 아이콘(원형 배경) + 카운트(우측 상단, 볼드) + 라벨(하단)
- 카드 색상:
  - 오늘: Blue `#007AFF`
  - 예정: Red `#FF3B30`
  - 전체: Dark Gray `#8E8E93`
  - 플래그: Orange `#FF9500`
  - 완료됨: Gray `#8E8E93`
- 카드 배경: 반투명 white/dark, `backdrop-filter: blur(20px)`, 둥근 모서리 (12px)
- hover 시 배경색 약간 어두워짐

**내 리스트 섹션**
- "내 리스트" 섹션 헤더 (작은 회색 텍스트, 대문자 스타일)
- 각 리스트 행: 색상 원형 아이콘(● 12px) + 리스트 이름 + 우측에 리마인더 카운트 (회색)
- 선택된 리스트: 배경색 `rgba(0,122,255,0.12)`, 텍스트 강조
- hover: 배경 `rgba(0,0,0,0.04)`
- 리스트 우클릭(컨텍스트 메뉴): 이름 변경, 색상 변경, 삭제

**하단 액션**
- "+ 리스트 추가" 버튼 — 사이드바 좌측 하단, 텍스트 + 아이콘

### 6.3 콘텐츠 영역 상세

**툴바**
- 리스트 이름: 좌측 상단, 볼드 큰 제목 (28px), 리스트 색상 적용
- 우측: 정렬 드롭다운 (`···` 메뉴 버튼)

**리마인더 행 (핵심 컴포넌트)**

Apple Reminders의 리마인더 행을 정밀하게 재현:

```
┌──────────────────────────────────────────────────┐
│  ○  장보기                                  ⓘ   │
│     우유, 빵, 계란                               │
│     📅 3월 21일   !! 높음   ⚑                   │
└──────────────────────────────────────────────────┘
```

- **체크박스(○)**: 리스트 색상의 빈 원(stroke), 24px
  - hover 시 원 내부에 연한 색상 fill
  - 클릭 시: 체크마크(✓) 애니메이션 → 0.5초 후 행이 fade out하며 완료 처리
  - 완료된 항목: 채워진 원 + 취소선 텍스트 (회색)
- **제목**: 본문 크기(15px), 검정/흰색
- **메모 미리보기**: 제목 아래 한 줄, 회색 작은 텍스트(13px), 1줄 truncate
- **메타 정보 행**: 메모 아래, 아이콘+텍스트 뱃지 형태
  - 마감일: 📅 아이콘 + 날짜 텍스트 (지난 날짜는 빨간색)
  - 우선순위: `!` 낮음, `!!` 보통, `!!!` 높음 (Blue 색상)
  - 플래그: 🚩 아이콘 (Orange)
  - 반복: 🔄 아이콘
- **디테일 버튼(ⓘ)**: 우측, 클릭 시 인라인 상세 편집 확장 또는 디테일 패널 오픈
- 행 사이 구분선: 얇은 회색 (0.5px), 체크박스 이후부터 시작 (Apple 스타일 인덴트 구분선)

**인라인 추가**
- 목록 하단 "+ 리마인더 추가" 클릭 → 새 빈 행이 나타나며 제목 입력 필드 포커스
- Enter로 즉시 생성, 연속 입력 가능
- Escape로 취소

**인라인 상세 편집 (Detail Popover)**

리마인더의 ⓘ 버튼 클릭 시, Apple Reminders와 동일한 인라인 상세 편집 영역 확장:

```
┌──────────────────────────────────────────┐
│  제목 입력 필드                            │
│  메모 입력 필드 (멀티라인)                  │
│  ────────────────────────────────────    │
│  URL          입력 필드                   │
│  ────────────────────────────────────    │
│  날짜          토글 (ON/OFF)              │
│               📅 2026년 3월 21일          │
│  시간          토글 (ON/OFF)              │
│               🕐 오전 9:00               │
│  ────────────────────────────────────    │
│  반복          안 함 ▾                    │
│  ────────────────────────────────────    │
│  우선순위       없음 ▾                    │
│  ────────────────────────────────────    │
│  리스트        개인 ● ▾                   │
│  ────────────────────────────────────    │
│  플래그         🚩 토글                   │
│  ────────────────────────────────────    │
│  하위 리마인더                             │
│   ○ 우유 사기                             │
│   ○ 빵 사기                              │
│   + 하위 리마인더 추가                     │
└──────────────────────────────────────────┘
```

- 각 필드는 Apple 설정 스타일의 행 레이아웃 (라벨 좌측 + 값/컨트롤 우측)
- 토글 스위치: iOS 스타일 (Green ON / Gray OFF)
- 날짜 선택: 캘린더 팝오버
- 드롭다운: Apple 스타일 팝오버 메뉴

### 6.4 스마트 리스트별 콘텐츠 뷰

**오늘 (Today)**
- 헤더: "오늘" + 날짜 서브텍스트 (예: "3월 21일 금요일")
- 리스트별로 그룹핑하여 표시 (섹션 헤더 = 리스트 이름 + 리스트 색상)

**예정 (Scheduled)**
- 날짜별 그룹핑 (예: "오늘", "내일", "3월 23일 일요일", ...)
- 각 날짜 섹션은 접기/펴기 가능
- 지난 날짜(overdue)는 상단에 빨간색 텍스트로 표시

**전체 (All)**
- 리스트별로 그룹핑
- 각 섹션 헤더: 리스트 색상 원 + 리스트 이름

**플래그 지정됨 (Flagged)**
- 플래그된 리마인더만 표시, 리스트별 그룹핑

**완료됨 (Completed)**
- 완료된 리마인더 목록
- 체크된 원형(●✓) + 취소선 제목 + 회색 텍스트
- 완료 날짜별 그룹핑 (오늘, 지난 7일, 지난 30일 등)

### 6.7 주요 인터랙션 상세

**컨텍스트 메뉴 (우클릭)**
- 사이드바 리스트: 이름 변경 / 색상 & 아이콘 변경 / 정렬 기준 / 리스트 삭제
- 리마인더 행: 인라인 편집 / 마감일 설정 (오늘/내일/주말) / 플래그 토글 / 우선순위 / 리스트 이동 / 삭제

**키보드 단축키**
| 단축키 | 동작 |
|--------|------|
| `⌘ + N` | 새 리마인더 추가 |
| `Enter` | 리마인더 생성 확인 / 다음 리마인더 인라인 추가 |
| `Escape` | 편집 취소 / 팝오버 닫기 |
| `Delete / Backspace` | 선택된 리마인더 삭제 |
| `⌘ + F` | 검색 필드 포커스 |
| `↑ / ↓` | 리마인더 목록 탐색 |
| `⌘ + .` | 플래그 토글 |

**빈 상태 (Empty State)**
- 리스트에 리마인더가 없을 때: 중앙에 연한 아이콘 + "리마인더 없음" 텍스트
- 검색 결과 없을 때: "검색 결과 없음"

### 6.8 반응형 레이아웃

**Desktop (≥1024px)**
- 사이드바 + 콘텐츠 영역 (2-column)
- 사이드바 리사이즈 가능 (드래그 핸들)

**Tablet (768px ~ 1023px)**
- 사이드바 오버레이 모드 (토글 버튼으로 열기/닫기)
- 콘텐츠 영역 전체 폭
- 상세 편집은 시트(bottom sheet) 또는 모달

**Mobile (<768px)**
- 네비게이션 스택 방식 (사이드바 → 리스트 → 상세)
- 하단 탭바 없음, 뒤로가기로 탐색
- 스와이프 제스처: 좌로 스와이프 → 삭제/플래그 액션
